# وثيقة تصميم واجهة المستخدم النهائية (UI Design) - MVP

**تاريخ الإنشاء:** 3 نوفمبر 2025
**الإصدار:** 1.0

## 1. مقدمة

هذه الوثيقة تقدم التصميم المرئي النهائي (UI Design) للمنتج الأولي القابل للتطبيق (MVP) لمنصة السوق البيطرية. التصميم مبني على وثيقة "تصميم المعلومات والنماذج الأولية (Wireframes)" ويهدف إلى إنشاء واجهة مستخدم حديثة، احترافية، وسهلة الاستخدام، مع التركيز على بناء الثقة وتسهيل العمليات التجارية للمستخدمين في القطاع البيطري.

## 2. نظام التصميم (Design System)

لضمان التناسق والكفاءة في التصميم والتطوير، سيتم اعتماد نظام تصميم متكامل يشمل المكونات التالية:

### أ. لوحة الألوان (Color Palette)

سيتم استخدام لوحة ألوان تعكس الاحترافية، الثقة، والنظافة، مع لمسة من الحيوية.

| اللون | الكود (Hex) | الاستخدام |
| :--- | :--- | :--- |
| **الأزرق الأساسي** | `#2A64F5` | الأزرار الرئيسية، الروابط، العناصر التفاعلية. يمثل الثقة والاحترافية. |
| **الأخضر الثانوي** | `#1DC9B7` | مؤشرات النجاح، الإشعارات الإيجابية، بعض الأزرار الثانوية. يمثل الصحة والحيوية. |
| **الرمادي الداكن** | `#333333` | النصوص الأساسية والعناوين. |
| **الرمادي المتوسط** | `#666666` | النصوص الفرعية والأوصاف. |
| **الرمادي الفاتح** | `#F5F7FA` | خلفيات الأقسام والبطاقات لتمييزها عن الخلفية الرئيسية. |
| **الأبيض** | `#FFFFFF` | الخلفية الرئيسية للموقع لضمان الوضوح وسهولة القراءة. |
| **الأحمر (للأخطاء)** | `#E74C3C` | رسائل الخطأ، تنبيهات الحذف. |
| **الأصفر (للتنبيهات)**| `#F1C40F` | رسائل التنبيه والتحذير. |

### ب. الخطوط (Typography)

سيتم استخدام خط واحد لضمان البساطة والوضوح.

*   **الخط:** **Cairo** (خط جوجل مجاني يدعم العربية والإنجليزية بشكل ممتاز).
*   **العناوين الرئيسية (H1, H2):** وزن **Bold**.
*   **العناوين الفرعية (H3, H4):** وزن **Semi-Bold**.
*   **النصوص الأساسية (Paragraph):** وزن **Regular**.

### ج. الأيقونات (Iconography)

سيتم استخدام مجموعة أيقونات حديثة وبسيطة من مكتبة **Feather Icons** أو **Font Awesome 6** لضمان الوضوح وسهولة الفهم.

### د. مكونات الواجهة (UI Components)

*   **الأزرار (Buttons):** تصميم واضح مع حواف دائرية قليلاً. الزر الأساسي باللون الأزرق، والزر الثانوي بتصميم مخطط (outline).
*   **حقول الإدخال (Input Fields):** تصميم بسيط ونظيف مع تسميات واضحة. عند التركيز (focus)، يظهر إطار باللون الأزرق.
*   **البطاقات (Cards):** تستخدم لعرض المنتجات والطلبات، مع حواف دائرية وظل خفيف لإعطاء إحساس بالعمق.
*   **الجداول (Tables):** تصميم نظيف وواضح لعرض البيانات، مع تظليل خفيف للصفوف عند المرور عليها (hover).

## 3. تصميم الشاشات (Screen Designs)

هنا وصف للتصميم المرئي لكل شاشة بناءً على الـ Wireframes.

### أ. الشاشات العامة (Public Screens)

1.  **الصفحة الرئيسية:**
    *   شريط تنقل علوي بسيط يحتوي على الشعار، روابط (المنتجات، تسجيل الدخول، تسجيل)، وزر "تسجيل" بارز باللون الأزرق.
    *   قسم رئيسي (Hero Section) يحتوي على صورة كبيرة عالية الجودة تعكس المجال البيطري، مع عنوان جذاب وزر "تصفح المنتجات".
    *   أقسام تعريفية بسيطة تعرض الميزات الرئيسية للمنصة.

2.  **صفحة التسجيل / تسجيل الدخول:**
    *   تصميم بسيط في المنتصف يحتوي على شعار المنصة، نموذج التسجيل/الدخول، وزر أساسي باللون الأزرق.
    *   روابط واضحة للتبديل بين التسجيل وتسجيل الدخول، ورابط "نسيت كلمة المرور".

### ب. شاشات المشتري (Buyer Screens)

1.  **لوحة التحكم:**
    *   قائمة تنقل جانبية تحتوي على روابط (الطلبات، الملف الشخصي، العناوين، إلخ).
    *   الجزء الرئيسي يعرض ملخصًا سريعًا: آخر الطلبات، إشعارات حديثة، ومنتجات مقترحة في بطاقات جذابة.

2.  **قائمة المنتجات:**
    *   تخطيط شبكي (Grid Layout) يعرض المنتجات في بطاقات. كل بطاقة تحتوي على صورة المنتج، الاسم، السعر، وزر "أضف إلى السلة".
    *   شريط جانبي يحتوي على فلاتر (الفئات، العلامات التجارية، السعر).

3.  **عربة التسوق وإتمام الطلب:**
    *   تصميم متعدد الخطوات وواضح (عربة التسوق -> عنوان الشحن -> الدفع -> تأكيد).
    *   استخدام اللون الأخضر لتأكيد النجاح في كل خطوة.

### ج. شاشات البائع (Seller Screens)

1.  **لوحة التحكم:**
    *   تصميم مشابه للوحة تحكم المشتري ولكن مع التركيز على بيانات البائع: ملخص المبيعات، أحدث الطلبات، والمنتجات الأكثر مبيعًا.

2.  **إدارة المنتجات:**
    *   جدول بيانات نظيف يعرض المنتجات مع خيارات (تعديل، حذف، إخفاء).
    *   زر "إضافة منتج جديد" بارز باللون الأزرق.

### د. شاشات المسؤول (Admin Panel)

1.  **لوحة التحكم:**
    *   تصميم احترافي وبسيط يركز على البيانات.
    *   استخدام رسوم بيانية (Charts) لعرض الإحصائيات الرئيسية (عدد المستخدمين، المبيعات، الطلبات).
    *   قائمة تنقل جانبية واضحة لكل أقسام الإدارة.

2.  **إدارة المستخدمين / المنتجات / الطلبات:**
    *   استخدام جداول بيانات قوية مع خيارات بحث وتصفية متقدمة.
    *   أزرار واضحة للإجراءات (موافقة، رفض، تعديل، حذف) مع استخدام الألوان المناسبة (الأخضر للموافقة، الأحمر للرفض/الحذف).

## 4. خاتمة

هذا التصميم يهدف إلى توفير تجربة مستخدم احترافية، موثوقة، وسهلة الاستخدام، مما يساهم في نجاح المنصة في السوق البيطري المتخصص. يجب استخدام هذه الوثيقة كمرجع أساسي لفريق تطوير الواجهة الأمامية (Frontend) لضمان تطبيق التصميم بدقة وتناسق.
